## O que esta tela faz
**Ordem de Faturamento (ODF)** — documento que **dirige a emissão das NFs** a partir do processo: define
configuração de faturamento, entrada/saída, CFOP, contrato e intermediador. É a ponte processo→NF (fase 50).
Referenciada no `imp021` (`importarDaOdf`, `ProcessoDespesasOdf`, `fatCheckPoint`).

## Como chegar
`home → /imp002` → **Ordem de Faturamento - Pesquisa** → Editar → `/imp002#/cadastro/{odfCod}`.

**ENTRADA IMPORTAÇÃO ENCOMENDA/CO3** (CFOP 3102/3949) · **REMESSA C&O** (5949) · **PRESTAÇÃO DE SERVIÇOS CONTA
E ORDEM** (5933) · **REMESSA ARMAZENAGEM** (5934) · **VENDA ENC. INTEREST** (6106). Unidade de Negócio típica:
**IMPORTAÇÃO CONTA E ORDEM** (C&O) — modelo de negócio Columbia (importa por conta e ordem de terceiros).

## Quirks
ODF gera as NFs de entrada (`com296`) e saída (`com297`). 111 endpoints. Configuração de ODF parametriza o
faturamento do processo. Entidade de itens = `ImpItensOrdFat`. ODF "Mãe/Filhote" = desdobramento de faturamento.
