## O que esta tela faz
**Títulos financeiros** (parcelas) dos documentos a pagar/receber. Cada documento fiscal/financeiro
(`com296`/`com297`/`com298`/`com299`) gera **títulos** (parcelas com vencimento e valor); é o que depois é
**baixado** via borderô ([`fin010`](fin010.md) a pagar / [`fin014`](fin014.md) a receber, fase 60).

## Como chegar
**Sub-tela** — os títulos abrem de dentro do documento financeiro (`com298`/`com299` → seção/aba Títulos;
o "RESUMO DOS TÍTULOS" aparece na aba Documentos do com298). ⬜ confirmar tela/abas ao vivo.

## Endpoints (swagger, 36 paths)
- `POST /api/com311/list` — lista de títulos (`CnxListRequest`).
- `POST /api/com311/agrupamento/list` — agrupamento de títulos.
- `POST /api/com311/varCambial/list` — **variação cambial por título** (cotejo taxa do título × baixa).

## Schema (campos-chave, a confirmar)
`titEspNumero` (nº/parcela), `titDtaVencimento`, `titMnyValor`, **`titFltTaxaMneg`** (taxa em moeda negociada),
`titVldStatus` (aberto/pago/permutado), `docCod`/`priCod` (vínculo doc/processo), `gerNum` (natureza).

## Quirks
Status **seed** (não confirmado ao vivo). `com311/varCambial` + `com308/financeiroAPagar/varCambial` são as
fontes nativas de **variação cambial por título** (fase 60→70). Ver `lifecycle/60-financeiro.md`.
