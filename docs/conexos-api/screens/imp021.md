## O que esta tela faz
O **Processo** é a entidade central (`priCod`) que amarra todo o ciclo de importação: liga pedido, invoice,
despacho, nacionalização, faturamento, financeiro e fechamento. Controla as **3 datas de fechamento**
(Operacional / Financeiro / Contábil) e a **taxa de câmbio** do processo. É o hub de navegação do ERP.

## Como chegar
`home → /imp021` abre **Processos - Pesquisa**. Preencha **Referência Externa** (ex.: `0021INX/25`) →
**Pesquisar** → selecione a linha → **Editar** → abre **Processos - Edição** em `/imp021#/cadastro/{filCod}/{priCod}`.

## Sub-telas em detalhe
### Despesas — rodapé → botão "Despesas"
`POST /api/imp021/DespesasProcesso/{priCod}` → `CnxListResponseImpProcessoDespesas`. Grid de despesas
rateadas. **É a fonte "ENCARGOS GERAIS > DESPESAS"** do MODELO DASHBOARD.
Colunas↔campos: Despesa=`impDesNome` · Valor=`pidMnyValormn` · Conta do Projeto=`ctpDesNome`/`ctpEspConta` ·
Forma de rateio=`pidVldFormaReteio` · Moeda=`moeEspNome` · Tipo=`tdsDesTipo`.
Valores reais (proc. 103): ICMS DIFAL R$ 1.646,19 · AFRMM R$ 1.666,18 · ENCARGOS FINANCEIROS R$ 5.766,05.

### Conta Corrente do Processo  ⭐ (DÉBITO × CRÉDITO — núcleo do fechamento)
Modal **"CONTA CORRENTE DO PROCESSO - IMP_040"** com o razão débito×crédito do processo (grid LANÇAMENTOS +
FILTROS + Crédito Total / Débito Total / **Diferença**). Botões: Detalhamento · Relatório · Fechar.
- Endpoint (versão leve, vazia): `POST /api/imp021/ContaCorrente` → `CnxListResponseViewCcPessoas`.
- Versão completa (com dados, título IMP_040): tela `imp040` (permissão `GET /api/permissoes/new/imp040`);
  endpoint de dados a pinar (rede instável na captura).
Colunas↔campos (`ViewCcPessoas`): Origem=`vwOrigem` · Lançamento=`cctDtaLcto` · Número Documento=`vwEspNumero` ·
Evento=`vwEvento` · **Valor Crédito**=`cctMnyCredito` · **Valor Débito**=`cctMnyDebito` · Vencimento=
`vwDtaVencimento` · Complemento=`cctMemComplemento` · (link doc `docCod`/`docTip`).
**Dados reais (proc. 103, 12 lançamentos):** crédito = VALOR TOTAL NF (172925/981/982), ICMS;
débito = VLR PAGO REF a forwarders/agentes (FFPV GROUP R$ 386.392,45/46 · MAERSK R$ 2.627,60 ·
AR SOLUÇÕES R$ 515,46 · SEARA R$ 10.259,75 · TIME LOG), CSRF. Evento de todos = FINANCEIRO.

### Validação — Mais Ações → Validação  ✅
Modal **"AVISOS"**: grid de validação do processo (coluna Descrição) com toggles **Exibir Avisos · Exibir
Erros** e botão Fechar. Lista avisos/erros de consistência do processo. Vazio no proc. 103 (sem pendências).

### Outras opções de Mais Ações (status honesto)

- **Responsáveis** — ⬜ 2 tentativas sem abrir modal (1ª congelou a página; 2ª não renderizou modal no proc.

  em processo com responsáveis atribuídos.
- **Checkpoints** — relaciona-se às seções CHECKPOINT do form (ver Layout) e a `fatCheckPoint`. ⬜ confirmar isoladamente.

### Encargos Gerais — NÃO está no Mais Ações do imp021
Encargos Gerais (`com017`) é acessado pelo Mais Ações da **NF** (`com297`/`com296`/`com298`), não do processo.

### Eventos (aba) — endpoint verificado ao vivo
`POST /api/imp021/ImpEventosProcesso/list/{priCod}` → `CnxListResponseImpEventosProcesso` (200 confirmado no
processo 103). É a **linha do tempo interna do processo**: cada linha = um evento/etapa com previsão e
realização. Row DTO `ImpEventosProcesso` (colunas↔campos):
- Ordem = `tevNumOrdem` · Evento = `evnDesNome` / Tipo = `tevEspDescricao` · Previsão = `prvDtaPrevisao` ·
  Resposta/Realização = `prvDtaResposta` · Responsável = `revEspNome` · Descrição = `prvMemDescricao` ·
  Confirmação = `usnDesNomeConfirmacao` · flags `ged`/`vldFat`/`vldDocLig` (doc/faturamento ligado).
> **Quirk:** a aba **só ativa com o processo totalmente carregado**, e o grid de eventos **trava o renderer**
> (screenshot expira) — provavelmente muitos eventos. Layout visual a capturar em sessão dedicada (ex.: paginar
> ou filtrar). Endpoint + schema confirmados; rótulos de coluna inferidos do DTO.

## Quirks / armadilhas
- Navegação por hash **não recarrega** — forçar `home → /imp021`.
- Sub-telas de despacho (`imp237`/`imp230`) **só abrem com contexto de processo** (drilldown), não por rota direta.
- Aba Eventos pode exigir o processo totalmente carregado para ativar.
