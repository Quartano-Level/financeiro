## O que esta tela faz
**Faturamento em Lote** — gera as notas fiscais de **vários processos/ODFs de uma vez** (batch), em vez de uma a
uma. É o atalho de produtividade do faturamento (fase 50): seleciona os ODFs ([`imp002`](imp002.md)) prontos e
dispara a emissão das NFs ([`com296`](com296.md) entrada / [`com297`](com297.md) saída), que depois são
transmitidas como NF-e por [`com319`](com319.md).

## Como chegar
⬜ A confirmar ao vivo — provável standalone (`home → /imp240`) ou via menu de faturamento.

## Endpoints (swagger)
- `POST /api/imp240/list` — lista de processos/ODFs faturáveis.
- `POST /api/imp240/gerar` — dispara a geração do lote de NFs.

## Quirks
Status **seed** (não confirmado ao vivo). Complementa o ODF (`imp002`): o ODF define *como* faturar; o
`imp240` fatura *em massa*. Ver `lifecycle/50-faturamento.md`.
